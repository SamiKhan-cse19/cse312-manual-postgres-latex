\section*{2. Introduction to PostgreSQL}
\defbox{PostgreSQL}{PostgreSQL is an open-source object-relational database management system. It is widely used for storing, organizing, and managing structured data.}

PostgreSQL supports standard SQL along with many advanced database features. In this lab, students
will use PostgreSQL to create a small relational database and practice basic database commands.

\noindent\textbf{Required software:}
\begin{itemize}
    \item PostgreSQL Server
    \item pgAdmin or psql command-line tool
\end{itemize}

\subsection*{2.2 Basic psql Navigation Commands}
Before using navigation commands, it is important to understand the database objects being
navigated.

\defbox{Database}{A database is an organized collection of related data. In PostgreSQL, a database contains schemas, tables, and other objects used to store and manage information for a particular application or purpose.}

\defbox{Schema}{A schema is a logical container inside a database. It groups related database objects such as tables, views, and functions, and helps organize large databases.}

\defbox{Table}{A table is a database object that stores data in rows and columns. Each table usually represents one entity, such as students, departments, courses, or employees.}

\noindent\textbf{Example:} In a \texttt{university\_lab} database, an \texttt{academics} schema may contain a \texttt{student} table.

After connecting to PostgreSQL, use the following commands to navigate databases, schemas, and
tables.

\section*{3. Relational Database Concepts}
A relational database stores data in tables. Each table represents an entity, and relationships
between tables are created using keys.

\subsection*{3.1 Tables, Rows, and Columns}
\defbox{Table}{A table is a collection of related data arranged in rows and columns. For example, a \texttt{student} table stores student information.}

\defbox{Column}{A column represents a specific attribute of a table, such as \texttt{student\_id}, \texttt{name}, or \texttt{email}.}

\defbox{Row}{A row represents one record in a table. For example, one row in the \texttt{student} table represents one student.}

\screenshotplaceholder{5cm}{Screenshot Placeholder: Example table showing rows and columns}

\subsection*{3.2 Common PostgreSQL Data Types}
Each column must have a data type. The data type defines what kind of value can be stored in the
column.

\begin{table}[H]
\centering
\renewcommand{\arraystretch}{1.4}
\begin{tabular}{>{\bfseries}p{4cm} p{8cm}}
\toprule
Data Type & Description \\
\midrule
INTEGER & Stores whole numbers. \\
SERIAL & Auto-incrementing integer, commonly used for primary keys. \\
VARCHAR(n) & Stores variable-length character strings with a maximum length. \\
TEXT & Stores variable-length text without a fixed maximum length. \\
NUMERIC(p, s) & Stores exact decimal values with precision and scale. \\
DATE & Stores calendar dates. \\
TIMESTAMP & Stores date and time values. \\
BOOLEAN & Stores true or false values. \\
\bottomrule
\end{tabular}
\caption{Frequently used PostgreSQL data types}
\label{tab:data_types}
\end{table}

\subsection*{3.3 Keys and Constraints}
\defbox{Key}{A key is a column or group of columns used to identify records or create relationships between tables.}

\defbox{Constraint}{A constraint is a rule applied to a table column or a group of columns to maintain data accuracy and integrity.}

\noindent Common keys and constraints include:
\begin{itemize}
    \item \texttt{PRIMARY KEY}: uniquely identifies each row in a table.
    \item \texttt{FOREIGN KEY}: links a column to the primary key of another table.
    \item \texttt{NOT NULL}: prevents a column from storing null values.
    \item \texttt{UNIQUE}: ensures all values in a column are different.
    \item \texttt{CHECK}: ensures values satisfy a specified condition.
    \item \texttt{DEFAULT}: assigns a value automatically when no value is provided.
\end{itemize}

\section*{4. Creating Tables with DDL Statements}
\defbox{DDL}{Data Definition Language is the subset of SQL used to define and manage database structures such as tables, schemas, and constraints.}

The \texttt{CREATE TABLE} statement creates a new table. The statement includes table name, column
names, data types, and constraints.

\subsection*{4.1 Creating a Simple Table}
\begin{verbatim}
CREATE TABLE department (
    dept_id SERIAL PRIMARY KEY,
    dept_name VARCHAR(100) NOT NULL UNIQUE,
    location VARCHAR(100)
);
\end{verbatim}

\screenshotplaceholder{5cm}{Screenshot Placeholder: Creating the department table}

\subsection*{4.2 Creating a Table with Different Data Types and Constraints}
\begin{verbatim}
CREATE TABLE student (
    student_id SERIAL PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    date_of_birth DATE,
    admission_year INTEGER CHECK (admission_year >= 2000),
    is_active BOOLEAN DEFAULT TRUE,
    dept_id INTEGER REFERENCES department(dept_id)
);
\end{verbatim}

\screenshotplaceholder{6cm}{Screenshot Placeholder: Creating the student table}

\subsection*{4.3 Creating a Table with a Foreign Key}
\begin{verbatim}
CREATE TABLE course (
    course_id SERIAL PRIMARY KEY,
    course_code VARCHAR(20) NOT NULL UNIQUE,
    course_title VARCHAR(150) NOT NULL,
    credit NUMERIC(3, 1) CHECK (credit > 0),
    dept_id INTEGER NOT NULL REFERENCES department(dept_id)
);
\end{verbatim}

\subsection*{4.4 Creating a Table with Composite Primary Key}
A composite primary key is formed using two or more columns. It is useful for junction tables that
represent many-to-many relationships.

\begin{verbatim}
CREATE TABLE enrollment (
    student_id INTEGER REFERENCES student(student_id),
    course_id INTEGER REFERENCES course(course_id),
    enrollment_date DATE DEFAULT CURRENT_DATE,
    grade VARCHAR(2),
    PRIMARY KEY (student_id, course_id)
);
\end{verbatim}

\section*{5. Modifying Tables with ALTER}
The \texttt{ALTER TABLE} statement modifies the structure of an existing table.

\subsection*{5.1 Adding a Column}
\begin{verbatim}
ALTER TABLE student
ADD COLUMN phone VARCHAR(20);
\end{verbatim}

\subsection*{5.2 Deleting a Column}
\begin{verbatim}
ALTER TABLE student
DROP COLUMN phone;
\end{verbatim}

\subsection*{5.3 Changing a Column Data Type}
\begin{verbatim}
ALTER TABLE student
ADD COLUMN contact_number VARCHAR(20);

ALTER TABLE student
ALTER COLUMN contact_number TYPE VARCHAR(30);
\end{verbatim}

\subsection*{5.4 Renaming a Column}
\begin{verbatim}
ALTER TABLE student
RENAME COLUMN contact_number TO mobile_number;
\end{verbatim}

\subsection*{5.5 Adding a Constraint}
\begin{verbatim}
ALTER TABLE enrollment
ADD CONSTRAINT grade_check
CHECK (grade IN ('A+', 'A', 'B+', 'B', 'C+', 'C', 'D', 'F'));
\end{verbatim}

\subsection*{5.6 More ALTER Commands for Further Exploration}
The following commands are useful in real database work. Students should explore them after
practicing the essential \texttt{ALTER TABLE} commands above and figure out what they do.

\begin{verbatim}
ALTER TABLE student RENAME TO university_student;

ALTER TABLE university_student
ALTER COLUMN email SET NOT NULL;

ALTER TABLE university_student
ALTER COLUMN is_active SET DEFAULT TRUE;

ALTER TABLE university_student
DROP CONSTRAINT IF EXISTS university_student_email_key;
\end{verbatim}

\section*{6. DROP and TRUNCATE Commands}
\subsection*{6.1 DROP TABLE}
The \texttt{DROP TABLE} command removes a table permanently. Use it carefully because the table
definition and its data are deleted.

\begin{verbatim}
DROP TABLE IF EXISTS temporary_table;
\end{verbatim}

\subsection*{6.2 TRUNCATE TABLE}
The \texttt{TRUNCATE} command removes all rows from a table but keeps the table structure.

\begin{verbatim}
TRUNCATE TABLE enrollment;
\end{verbatim}

If another table references the table, \texttt{CASCADE} may be needed. Use it carefully.

\begin{verbatim}
TRUNCATE TABLE enrollment CASCADE;
\end{verbatim}
